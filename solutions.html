QMTS

Overview

This system allows several remote devices or hosts to securely communicate as though they are connected by a single IP network with uncomplicated routing. This is achieved through a central hub, the Quick Route Managing Service (QRMS), to which all clients connect. The QRMS is responsible for address allocation and routing between all the links as well as security functions.

In cases where a device has more than one connection, the QRMS also evaluates and aggregates individual link quality to maximise throughput and availability. Connections to the QRMS can be made over any public or private TCP/IP network and are insensitive to intermediate NAT and firewall operations over which hosts may have no direct control.

For security, the QRMS link protocol also encrypts all data using link-unique random keys agreed on a per-connection basis through Diffie-Hellman exchange (ephemeral keying). Optional authentication of both the client and QRMS can be performed by exchanging cryptographic hashes which incorporate pre-shared key material.

Quad Modem Telemetry System (QMTS) devices are one type of QRMS client and are designed to utilise up to 4 wireless modems to provide exceptional link reliability and throughput when connecting to the central QRMS. The aggregate link may then be used by applications running either directly on the QMTS device, or by further hosts utilising the QMTS as an IP gateway on a sub-network.

Other machines or servers may connect to the QRMS using provided software-only clients and services, available both for Windows and Linux.
